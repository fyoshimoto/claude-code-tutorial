Section 01

Introduction to Claude Code

Claude Code is Anthropic's official agentic coding assistant that runs directly in your terminal. Unlike chat-based AI tools, Claude Code can read your entire codebase, edit files, run shell commands, interact with Git, and autonomously complete multi-step engineering tasks — all from a single CLI.

🔍

Codebase Aware

Reads, searches, and understands your entire project without manual copy-paste.

✏️

File Editing

Creates, edits, and deletes files using precise surgical operations.

🚀

Shell Execution

Runs tests, builds, linters, git commands, and arbitrary shell scripts.

🔗

MCP Servers

Connects to external services (GitHub, Jira, databases) via the Model Context Protocol.

🤖

Multi-Agent

Spawns sub-agents in parallel for large, complex tasks — orchestrated automatically.

🔒

Permission Modes

Fine-grained control over what Claude can do — from safe-default to full-auto.

ℹ️

Claude Code sends your code and conversation to Anthropic's API. Code is not used to train models by default. Review the Privacy Policy and set ANTHROPIC_API_KEY before use.

Prerequisites

  • Node.js 18+ — node --version to check
  • npm / npx — bundled with Node.js
  • An Anthropic API Key — get one at console.anthropic.com
  • Windows users: use WSL 2 (Ubuntu recommended)

Install via npm

Install globallybash

npm install -g @anthropic-ai/claude-code

# Verify installation

Set your API Key

# Reload your shell
source ~/.bashrc

Section 03

First Run

  1. Navigate to your project 
    cd /path/to/your/project
  2. Launch Claude Code 
    claude

    An interactive REPL opens. Claude automatically detects your project type, reads key files, and prepares to assist.

  3. Ask your first question 
    > Explain the structure of this project
> What does the main function in src/index.ts do?
> Fix the failing test in tests/auth.test.js
  4. Review and approve tool calls

    Claude will ask permission before editing files or running commands (in default mode). You can approve y, deny n, or say "always allow" to auto-approve future calls.

Non-interactive / one-shot mode

Print modebash
# Run a single prompt and exit
claude -p "What is the purpose of package.json?"

# Pipe input
cat src/app.ts | claude -p "Summarize this file"
Section 04

How Claude Code Works

Claude Code operates as an agentic loop:

  1. You send a natural-language message
  2. Claude decides which tools to call (read file, run command, edit file…)
  3. Each tool call is shown to you; you approve or deny
  4. Results feed back into Claude's context
  5. Claude continues until the task is done or asks a follow-up

Built-in Tools

ToolWhat it does
ReadRead a file (with line numbers)
WriteCreate or overwrite a file
EditSurgical string replacement inside a file
BashExecute a shell command
GlobFind files by pattern (**/*.ts)
GrepSearch file contents by regex
TaskSpawn a sub-agent for complex subtasks
WebFetchFetch and summarise a URL
WebSearchSearch the web for up-to-date info
💡

Claude chooses the most efficient tool for each step automatically. You don't need to specify which tool to use — just describe what you want.

Section 05

Interactive Mode

The default claude command opens an interactive REPL where you converse with Claude across multiple turns, building up context progressively.

Useful flags

CLI flagsbash
# Resume the most recent conversation
claude --continue

# Resume a specific session by ID
claude --resume <session-id>

# Start with a system prompt override
claude --system-prompt "You are a strict TypeScript reviewer"

# Specify a model
claude --model claude-opus-4-6

# Verbose output (see all tool inputs/outputs)
claude --verbose

# Maximum context turns before auto-compacting
claude --max-turns 50

Context Management

Claude maintains conversation context within a session. When context grows large, use /compact to summarise and free space without losing key information. Use /clear to start fresh.

Section 08

Reading & Understanding Code

Claude can read your entire project. Here are effective prompts for code comprehension:

Example promptsprompt
# Understand project structure
Explain the overall architecture of this project

# Trace a data flow
How does a user login request flow from the route handler to the database?

# Understand a specific file
Explain what src/services/auth.service.ts does and its dependencies

# Find where something is defined
Where is the `parseConfig` function defined and what does it return?

# Identify dependencies
What external packages does this project depend on and why?

Providing context efficiently

  • Use file paths in your prompt: "Look at src/api/routes.ts"
  • Use line numbers: "Starting at line 42 in auth.ts"
  • Reference function/class names directly: "The UserService class"
  • Let Claude search itself: "Find all usages of the `deprecated` decorator"
Section 09

Editing Code

Claude uses surgical Edit operations (not full rewrites) to modify files — this makes diffs reviewable and minimises unintended changes.

Editing promptsprompt
# Fix a bug
The `calculateTotal` function in src/cart.ts returns NaN when the cart is empty. Fix it.

# Add a feature
Add an optional `timeout` parameter (default 5000ms) to the `fetchUser` function.

# Refactor
Refactor the nested callbacks in src/legacy.js to use async/await.

# Add tests
Write Jest unit tests for the `validateEmail` function in utils/validators.ts.

# Remove dead code
Remove the commented-out `legacyParser` function and its imports.

Plan mode

For large changes, ask Claude to plan first before touching any files:

Plan before editprompt
Before writing any code, give me a detailed plan for migrating
the auth system from JWT to session-based cookies.
⚠️

Always review Claude's edits before approving, especially for Write (full file overwrite) vs Edit (surgical replacement). Destructive operations will always ask for confirmation.

Section 12

Memory & CLAUDE.md

Claude Code has two types of persistent memory:

1. CLAUDE.md — Project Instructions

Place a CLAUDE.md file in your project root. Claude reads it automatically at the start of every session. Use it to encode:

  • Project architecture overview
  • Coding conventions and style rules
  • Which commands to use (bun vs npm, vitest vs jest)
  • Things Claude should always or never do
  • Environment setup notes
CLAUDE.md examplemarkdown
# Project: MyApp

## Tech Stack
- Runtime: Bun (always use `bun` not `npm` or `node`)
- Framework: Hono
- Database: PostgreSQL via Drizzle ORM
- Testing: Vitest

## Coding Rules
- All functions must be typed with explicit return types
- Use `const` over `let` unless reassignment is necessary
- No `any` types — use `unknown` and narrow with guards
- Tests live alongside source files as `*.test.ts`

## Never
- Commit secrets or API keys
- Use `console.log` in production code (use the logger utility)
- Skip type checks or linting

2. Auto Memory (~/.claude/projects/…)

Claude maintains per-project memory files in ~/.claude/projects/<path>/memory/. Key points:

  • MEMORY.md is auto-loaded each session (keep it under 200 lines)
  • Create topic files (e.g., patterns.md, debugging.md) for detailed notes
  • Ask Claude to "remember" something and it will save it
  • Ask Claude to "forget" something and it will remove the entry
Section 13

MCP Servers

The Model Context Protocol (MCP) lets Claude connect to external services and databases via standardised server plugins. Claude Code ships with support for several built-in MCP tools and lets you add your own.

What you can connect to

  • GitHub (issues, PRs, code search)
  • Jira / Linear (tickets, sprints)
  • PostgreSQL / MySQL / SQLite (run queries)
  • Slack (read channels, send messages)
  • Puppeteer / Playwright (web automation)
  • Custom REST APIs (via a small wrapper server)

Configure MCP in settings

~/.claude/settings.jsonjson
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_..."
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://localhost/mydb"]
    }
  }
}
✅

Project-level MCP config goes in .claude/settings.json (relative to project root). This can be committed to share MCP setup with your team without committing secrets (use env vars).

Section 14

Hooks

Hooks are shell commands that run automatically in response to Claude Code events. Configure them in ~/.claude/settings.json:

Hook configurationjson
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{
          "type": "command",
          "command": "echo 'Running: $CLAUDE_TOOL_INPUT' >> ~/.claude/audit.log"
        }]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [{
          "type": "command",
          "command": "prettier --write $CLAUDE_TOOL_INPUT_FILE_PATH 2>/dev/null"
        }]
      }
    ]
  }
}

Hook events

EventWhen it fires
PreToolUseBefore any tool call executes
PostToolUseAfter a tool call completes
UserPromptSubmitWhen you press Enter on a prompt
StopWhen Claude finishes a response

Common use cases: auto-format on save, audit logging, running linters after edits, sending Slack notifications on task completion.

Section 15

Permissions & Trust

Claude Code has three permission modes controlling how much it can do autonomously:

ModeBehaviourUse when
Default Asks approval for file writes and shell commands Day-to-day coding
Auto-approve Pre-approves specific tools or patterns Trusted, repetitive tasks
Headless / CI Fully non-interactive, all safe ops allowed Automated pipelines

Configure allow/deny lists

settings.jsonjson
{
  "permissions": {
    "allow": [
      "Bash(npm run test:*)",
      "Bash(git diff*)",
      "Read(**)"
    ],
    "deny": [
      "Bash(rm -rf*)",
      "Bash(git push --force*)"
    ]
  }
}

Headless Mode & CI/CD

Claude Code can run entirely non-interactively, making it perfect for automated pipelines:

Headless flagsbash
# Non-interactive, no permission prompts
claude -p "Review this PR for security issues" \
  --output-format json \
  --max-turns 10

# Pipe output to another tool
claude -p "Generate a changelog from git log" | tee CHANGELOG.md

GitHub Actions example

.github/workflows/claude-review.ymlyaml
name: Claude Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code
      - name: Run Code Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p "Review the changes in this PR for bugs, \
          security issues, and code quality. Output a markdown report." \
          --output-format text >> $GITHUB_STEP_SUMMARY
Section 17

Prompting Tips

Be specific about context

Bad vs Goodprompt
✗ Bad:  Fix the bug
✓ Good: Fix the NullPointerException in UserService.createUser()
         that occurs when the email field is null.
         The test is in tests/user.service.test.ts line 87.

Constrain the scope

✗ Bad:  Refactor the codebase
✓ Good: Refactor only the parseConfig() function in src/config.ts
         to use a Zod schema for validation. Don't touch other files.

Ask for a plan first on big tasks

✓ Good: Before writing any code, give me a step-by-step plan
         for adding OAuth2 login with Google. I'll approve the plan
         before you start editing files.

Useful phrases

  • "Don't touch any other files" — keeps changes focused
  • "Explain your reasoning" — forces more careful analysis
  • "Show me the diff before applying" — review before commit
  • "Run the tests after each change" — test-driven iteration
  • "Remember X for future sessions" — saves to memory
  • "What would break if I changed Y?" — impact analysis

Multi-Agent Workflows

For large, complex tasks Claude Code can spawn sub-agents in parallel using the Task tool. This is handled automatically — you just describe the big goal.

Multi-agent prompt exampleprompt
Migrate the entire codebase from JavaScript to TypeScript.
Work on different modules in parallel:
- src/api/
- src/services/
- src/utils/
Run type-check after each module is done. Report summary at the end.

Worktrees for isolation

Claude can create Git worktrees to isolate experimental changes from the main working tree:

Worktree promptprompt
Work in a new worktree so I can review the changes before merging.
Implement the new caching layer, run tests, then give me the branch name.
💡

Each sub-agent gets its own context window but shares the same filesystem. Claude coordinates between agents automatically, including merging results and resolving conflicts.

Section 19

Security Best Practices

  • Never commit API keys — Claude will warn you if it detects secrets in staged files
  • Review shell commands — always read what Claude plans to run before approving Bash calls
  • Use deny lists — block dangerous patterns (rm -rf, DROP TABLE) in settings
  • Restrict MCP scopes — give each MCP server the minimum required permissions
  • Audit logs via hooks — use PreToolUse hooks to log all tool calls for review
  • Use worktrees — isolate large AI-driven changes before they touch your main branch
  • Watch for prompt injection — if tool results contain suspicious instructions, Claude will flag them
⚠️

Treat Claude Code with the same trust level as a capable junior developer with access to your terminal. Review its work, especially on destructive operations and anything touching infrastructure or secrets.

Helpful security-focused prompts

Security review promptsprompt
# OWASP review
Review src/api/ for OWASP Top 10 vulnerabilities and report with severity

# Secret scanning
Scan the codebase for any hardcoded secrets, API keys, or passwords

# Dependency audit
Run npm audit and explain which vulnerabilities are critical and how to fix them
Section 20

GitHub Integration

Claude Code Action is an official GitHub Action that brings Claude directly into your GitHub workflow. It lets you invoke Claude by mentioning @claude in issue comments, pull request reviews, or via automated triggers — and Claude can read code, implement fixes, and push commits back to the branch automatically.

ℹ️

The action is open-source at anthropics/claude-code-action. It runs Claude Code in a GitHub Actions runner with full access to your repository.

What Claude can do inside GitHub

  • Answer questions about the codebase in issue or PR comments
  • Review pull requests and leave detailed feedback
  • Implement changes from an issue and open a pull request automatically
  • Fix failing CI when asked via a comment
  • Explain code by reading files and responding inline

Setup: Add the workflow file

Create .github/workflows/claude.yml in your repository:

.github/workflows/claude.ymlyaml
name: Claude Code
on:
  issue_comment:
    types: [created]
  pull_request_review_comment:
    types: [created]
  issues:
    types: [assigned]
  pull_request_review:
    types: [submitted]

jobs:
  claude:
    if: |
      (github.event_name == 'issue_comment' && contains(github.event.comment.body, '@claude')) ||
      (github.event_name == 'pull_request_review_comment' && contains(github.event.comment.body, '@claude')) ||
      (github.event_name == 'pull_request_review' && contains(github.event.review.body, '@claude')) ||
      (github.event_name == 'issues' && github.event.action == 'assigned')
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
      issues: write
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}

Add your API key as a secret

  1. Go to your repository on GitHub → Settings → Secrets and variables → Actions
  2. Click New repository secret, name it ANTHROPIC_API_KEY, and paste your key from console.anthropic.com
  3. Commit the workflow file and push — the action will be active immediately.

Triggering Claude in GitHub

Once installed, mention @claude in any issue or PR comment:

Example GitHub commentsmarkdown
# Ask a question
@claude Can you explain how the auth middleware works?

# Request an implementation
@claude Please implement the changes described in this issue and open a PR.

# Request a code review

Using the GitHub App (recommended)

Instead of a PAT, you can install the official Claude GitHub App which handles authentication automatically and grants fine-grained repository permissions:

  1. Visit the Claude GitHub App page
  2. Click Install and select the repositories to grant access
  3. Use use_bedrock or use_vertex options in the action for enterprise setups
App-based auth in workflowyaml
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          github_token: ${{ secrets.GITHUB_TOKEN }}
          # Optional: restrict which users can trigger Claude
          allowed_users: "octocat,other-teammate"
          # Optional: override the trigger phrase
          trigger_phrase: "@claude"

The action cannot modify .github/workflows/ files due to GitHub App permission restrictions. It also cannot force-push to protected branches or approve its own pull requests.
